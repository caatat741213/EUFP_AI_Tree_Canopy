## Assignment 1 vs. Assignment 2

* `Assignment_1.ipynb` focuses on the macro-level urban forestry DSS workflow, including canopy coverage analysis, heat-risk interpretation, socio-economic equity mapping, and a simple linear regression model for overall regional budget forecasting.
* `Assignment_2.ipynb` extends the DSS framework into a micro-level multi-dimensional asset risk prioritization engine across the full tri-city region (Waterloo, Kitchener, Cambridge). It delivers a head-to-head rapid prototyping evaluation comparing engineering paths **'Without Generative AI Tools'** (manual ETL and shallow Scikit-Learn Random Forest) versus **'With Generative AI Tools'** (automated pipeline orchestration, probabilistic climate data augmentation, and advanced PyTorch Neural Networks), successfully establishing a 100% closed-loop operational funding allocation.

## What The Notebooks Do

### 📘 Part 1: Assignment 1 Core Workflow

The primary pipeline in `Assignment_1.ipynb` establishes the environmental baseline:

1. Merging Waterloo and Kitchener street tree inventory datasets.
2. Converting flat tree inventories into spatial geometric data structures.
3. Spatial joining of tree distributions with Statistics Canada census tract boundaries.
4. Structuring macro DSS indicators: total asset counts, critical trees (`DBH > 45cm`), estimated canopy density, and ESPA proximity flags.
5. Generating localized diagnostic visualizations for urban heat island mitigation, canopy equity planning, and bulk budget boundaries.

### 🚀 Part 2: Assignment 2 Twin-Track Prototyping Pipeline

The advanced optimization pipeline in `Assignment_2.ipynb` runs a head-to-head comparison to solve operational resource constraints:

1. **Track A: Without Generative AI Tools (Traditional Manual Path)**
* Hardcoding manual, rigid data cleaning functions to align conflicting Waterloo and Kitchener data schemas.
* Engineering a baseline rule-set variable to substitute missing target values: $\text{Risk} = (\text{DBH} \times 0.7) + ((20 - \text{Canopy}) \times 1.2)$.
* Training a shallow Scikit-Learn `RandomForestRegressor` and executing local hyperparameter optimization via 2-fold cross-validation grid search.


2. **Track B: With Generative AI Tools (GenAI-Accelerated Path)**
* Deploying an abstract, automated multi-city mapping script (`ai_generated_cleaner`) to integrate the previously missing third municipality (**Cambridge**) seamlessly within seconds.
* Executing a probabilistic simulation engine for data augmentation, synthesizing 1,000 high-fidelity learning targets ($\text{Windstorm Damage Index}$) by integrating the empirical climate regression slope ($-0.24^\circ\text{C}$) paired with a stochastic Gaussian noise term $\mathcal{N}(0, 4.0)$ to mirror real-world windstorms.
* Coding and optimizing an advanced multi-layer Deep Learning Neural Network (PyTorch MLP) with automated tensor casting to map non-linear microclimatic stresses.


3. **Auditing & Analytics Integration**
* Logging human development time across both tracks to generate professional velocity analytics.

## Model Summaries & Mathematical Frameworks

### 1. Macro Budget Estimation Model (Assignment 1)

```text
Annual Maintenance Cost = 60,650 + 1,197 * Critical Trees

```

* **Intercept:** $60,650 CAD (Base municipal operational overhead)
* **Cost Per Unit:** $1,197 CAD (Proactive contractual trimming fee per critical asset)
* **R-squared:** `0.9996` (Validates highly reliable bulk fiscal forecasting)

### 2. Micro Risk Engine Formulations (Assignment 2 Dual-Track)

* **Track A (Without GenAI Tools - Rigid Deterministic Baseline):**

$$\text{Calculated Risk Score} = (\text{Tree Diameter}_{\text{cm}} \times 0.7) + ((20 - \text{Canopy Cover}_{\%}) \times 1.2)$$



*Captures structural burden while compensating for percentage data scale boundaries utilizing a 1.2 scaling multiplier.*
* **Track B (With GenAI Tools - Probabilistic Simulation & Neural Inputs):**

$$\text{Simulated Surface Temp} = 36.5 - (0.24 \times \text{Canopy Cover}_{\%})$$


$$\text{Windstorm Damage Index} = (\text{Tree Diameter}_{\text{cm}} \times 0.45) + (\text{Simulated Surface Temp} \times 0.75) + \mathcal{N}(0, 4.0)$$



*Integrates the -0.24°C microclimatic slope from Assignment 1 and applies a standard deviation 4.0 Gaussian normal error to represent windstorm stochasticity, training a 4-variable input PyTorch MLP.*

## How To Run
1. Clone the repository:

```bash
git clone https://github.com/caatat741213/Enhancing-Urban-Forestry-Planning-with-an-AI-Powered-Tree-Canopy-DSS.git
```

2. Create and activate a virtual environment:
```bash
#Windows
python -m venv .venv
.venv\Scripts\activate

# macOS/Linux:
python3 -m venv .venv
source .venv/bin/activate
```

3. Install the required packages:

```bash
pip install -r requirements.txt
```
4. Verify all raw source files are correctly placed within the `./datasets/` root directory.

5. Open and run `Assignment_1.ipynb` first to execute the baseline geospatial joins and generate the macro regression parameters.
6. Execute `Assignment_2.ipynb` to run the double-track simulation, train the PyTorch MLP deep learning network, and output the efficiency comparison graphics directly to the `./img/` folder.
